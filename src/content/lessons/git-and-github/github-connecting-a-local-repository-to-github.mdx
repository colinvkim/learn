After creating commits locally, the next step is connecting the repository to GitHub.

## Start by creating the repository on GitHub

In the GitHub web UI, create a new repository.

That gives GitHub a remote destination for your local repository.

At this stage, you usually want:

- the repository name
- the remote URL GitHub provides

## `git remote add origin` connects the local repository to the remote

To add the GitHub repository as your main remote:

```bash
git remote add origin https://github.com/OWNER/REPO.git
```

This does not upload your commits yet. It only records the remote connection in the local repository configuration.

### SSH vs HTTPS URLs

GitHub supports two authentication methods:

- **HTTPS** (`https://github.com/OWNER/REPO.git`) — works everywhere but may require entering credentials each time unless you use a credential helper
- **SSH** (`git@github.com:OWNER/REPO.git`) — uses SSH keys for authentication, recommended for regular contributors

To set up SSH authentication:

1. Generate an SSH key if you do not have one: `ssh-keygen -t ed25519 -C "you@example.com"`
2. Add the public key to your GitHub account under Settings > SSH and GPG keys
3. Use the SSH URL when connecting:

```bash
git remote add origin git@github.com:OWNER/REPO.git
```

SSH is the recommended method for regular use because it avoids repeated credential prompts.

## An easier way: GitHub CLI

The GitHub CLI (`gh`) is an official command-line tool that handles authentication, SSH key setup, and Git configuration in a single interactive command.

If you prefer not to manage SSH keys and Git remotes manually, `gh` does most of it for you.

### Installing GitHub CLI

On macOS, install with Homebrew:

```bash
brew install gh
```

On other platforms, see the [GitHub CLI installation guide](https://github.com/cli/cli#installation).

### Authenticating with `gh auth login`

Run:

```bash
gh auth login
```

This starts an interactive flow that:

1. Asks you to choose a protocol (**SSH** or **HTTPS**)
2. If you choose SSH, it detects existing SSH keys and offers to generate and upload a new one if none exist
3. Opens your browser to authorize your GitHub account
4. Configures Git to use the chosen protocol automatically

After `gh auth login` completes:

- Your GitHub account is authenticated in the terminal
- Your SSH key (if you chose SSH) is generated, uploaded, and configured
- Git is configured to use that key for pushes and pulls

<Tip title="gh auth login replaces several manual steps">
  <p>Without GitHub CLI, setting up Git auth involves generating an SSH key, adding it to GitHub, and configuring the remote URL manually. <code>gh auth login</code> does all of this interactively. If you have already set up SSH manually, you can skip <code>gh</code> — but it is useful for managing multiple accounts and checking your authentication status with <code>gh auth status</code>.</p>
</Tip>

### Connecting a repository with `gh repo create`

If you have a local repository and want to create the GitHub remote and push in one step:

```bash
gh repo create
```

This interactively asks for the repository name, visibility, and whether to push an existing local repository. It creates the remote, pushes your commits, and even offers to open the repository in your browser.

If you prefer the manual approach, the rest of this lesson covers how to connect and push without `gh`.

## `git push -u origin <branch>` sends the local branch to GitHub

A common first push looks like this:

```bash
git push -u origin main
```

Here:

- `origin` is the remote name
- `main` is the branch being pushed
- `-u` tells Git to remember the upstream tracking relationship

This does two things:

- pushes the local `main` branch to the `origin` remote
- sets the upstream tracking relationship for that branch

That upstream link is why later commands can often use shorter forms such as `git push` and `git pull`.

## A clean first-push flow

<Steps>
  <li>Create the repository on GitHub.</li>
  <li>Copy the repository URL.</li>
  <li>In the local repository, run <code>git remote add origin ...</code>.</li>
  <li>Push the current branch with <code>git push -u origin main</code>.</li>
</Steps>

## What changed locally and remotely

After this flow:

- the local repository knows about a remote named `origin`
- the local branch has been uploaded to GitHub
- GitHub now has the commit history that existed locally

<Note title="Adding a remote is not the same as pushing">
  <p><code>git remote add origin ...</code> only records the remote connection. The repository history is not uploaded until you run <code>git push</code>.</p>
</Note>
