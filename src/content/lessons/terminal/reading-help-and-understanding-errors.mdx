One of the most important terminal skills is not memorization. It is reading.

## `--help` is often the fastest way to learn a command

Many commands provide built-in help text.

Examples:

```bash
git --help
git commit --help
pnpm --help
```

Many tools also support a shorter help flag:

```bash
git commit -h
```

That help text usually explains:

- what the command does
- which arguments it expects
- which flags are available

## Error messages usually describe a real mismatch

An error message often means one of these things:

- the command name is wrong
- the path is wrong
- the current directory is wrong
- the command is missing a required argument
- a file or directory does not exist

That is why blindly retrying a command is often slower than reading the message carefully.

## A useful debugging order

When a terminal command fails, check these first:

1. Am I in the right directory?
2. Is the path correct?
3. Did I miss a flag or argument?
4. Does the file or directory actually exist?

## Help plus inspection is usually enough

In beginner workflows, a good recovery loop is:

1. check the current directory with `pwd`, or `cd` with no argument in Command Prompt
2. inspect the directory with `ls` or `dir`
3. read the error message carefully
4. use `--help` or `-h` if the command syntax is unclear

<Note title="Most terminal errors are not mysterious">
  <p>They usually point to a mismatch between what you asked for and what exists right now. Treat them as information, not as signs that the terminal is inherently fragile.</p>
</Note>
