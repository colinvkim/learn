This page is a quick reference for the terminal commands taught in the earlier lessons.

The exact command can vary by shell. Unix-style shells, PowerShell, and Command Prompt share the same core ideas, but not always the same command names.

## Show the current directory

<ShellCommandSet
  unix={`pwd`}
  powershell={`Get-Location`}
  cmd={`cd`}
/>

## List files and directories

<ShellCommandSet
  unix={`ls
ls -la`}
  powershell={`Get-ChildItem
Get-ChildItem -Force`}
  cmd={`dir
dir /a`}
/>

## Change directories

<ShellCommandSet
  unix={`cd src
cd ..
cd /absolute/path/here`}
  powershell={`Set-Location src
Set-Location ..
Set-Location C:\\absolute\\path\\here`}
  cmd={`cd src
cd ..
cd C:\\absolute\\path\\here`}
/>

## Create directories

<ShellCommandSet
  unix={`mkdir notes
mkdir -p src/content/lessons`}
  powershell={`New-Item -ItemType Directory notes
New-Item -ItemType Directory src/content/lessons -Force`}
  cmd={`mkdir notes
mkdir src\\content\\lessons`}
/>

## Create an empty file

<ShellCommandSet
  unix={`touch README.md`}
  powershell={`New-Item -ItemType File README.md`}
  cmd={`type nul > README.md`}
/>

## Copy files or directories

<ShellCommandSet
  unix={`cp README.md README.backup.md
cp -r src src-backup`}
  powershell={`Copy-Item README.md README.backup.md
Copy-Item src src-backup -Recurse`}
  cmd={`copy README.md README.backup.md
xcopy src src-backup /E /I`}
/>

## Move or rename

<ShellCommandSet
  unix={`mv draft.txt final.txt
mv notes.txt docs/notes.txt`}
  powershell={`Move-Item draft.txt final.txt
Move-Item notes.txt docs/notes.txt`}
  cmd={`move draft.txt final.txt
move notes.txt docs\\notes.txt`}
/>

## Remove files or directories

<ShellCommandSet
  unix={`rm notes.txt
rm -r old-folder`}
  powershell={`Remove-Item notes.txt
Remove-Item old-folder -Recurse`}
  cmd={`del notes.txt
rmdir /s old-folder`}
/>

## Read built-in help

```bash
git --help
git commit -h
pnpm --help
```

<Tip title="Break commands into parts">
  <p>When a command looks confusing, separate it into the command name, its arguments, and its flags. That usually makes the line much easier to understand.</p>
</Tip>

<Warning title="Read rm commands carefully">
  <p>Before using <code>rm</code>, confirm the path and the flag set. Removal is the place where terminal mistakes are most likely to be expensive.</p>
</Warning>
