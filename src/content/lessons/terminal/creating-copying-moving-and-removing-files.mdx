Once you can move around the filesystem, the next step is changing it.

## Creating directories with `mkdir`

To create a directory:

```bash
mkdir notes
```

This command works broadly across Unix-style shells, PowerShell, and Command Prompt.

To create nested directories in one step:

```bash
mkdir -p src/content/lessons
```

## Creating empty files with `touch`

To create an empty file or update its timestamp:

```bash
touch README.md
```

`touch` is common in Unix-style shells, but not in Command Prompt. On Windows, many beginners create files in an editor or use PowerShell commands instead. The important idea is that terminals can create files, even if the exact command differs by shell.

## Copying with `cp`

To copy a file:

```bash
cp README.md README.backup.md
```

To copy directories recursively:

```bash
cp -r src src-backup
```

Windows equivalents vary by shell:

- PowerShell commonly uses `Copy-Item`
- Command Prompt commonly uses `copy` for files and `xcopy` or `robocopy` for larger directory operations

## Moving or renaming with `mv`

To move a file:

```bash
mv notes.txt docs/notes.txt
```

To rename a file:

```bash
mv draft.txt final.txt
```

On Windows:

- PowerShell commonly uses `Move-Item`
- Command Prompt commonly uses `move`

## Removing with `rm`

To remove a file:

```bash
rm notes.txt
```

To remove a directory and its contents:

```bash
rm -r old-folder
```

On Windows:

- PowerShell commonly uses `Remove-Item`
- Command Prompt commonly uses `del` for files and `rmdir` for directories

<Warning title="rm deserves extra care">
  <p>Destructive terminal commands such as <code>rm</code>, <code>Remove-Item</code>, <code>del</code>, and <code>rmdir</code> deserve extra care. They often delete directly rather than sending content to a graphical trash folder. Read the path carefully before running them.</p>
</Warning>

## A practical rule

Creation and copying are usually reversible.

Removal is not always easy to undo.

That is why terminal users often move more slowly around `rm` than around `mkdir`, `touch`, `cp`, or `mv`.
