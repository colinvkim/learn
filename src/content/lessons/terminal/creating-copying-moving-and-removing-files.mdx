Once you can move around the filesystem, the next step is changing it.

## Creating directories with `mkdir`

To create a directory:

<CommandBlock code="mkdir notes" language="bash" metaLabel="Terminal" />

This command works broadly across Unix-style shells, PowerShell, and Command Prompt.

To create nested directories in one step:

<CommandBlock
  metaLabel="Terminal"
  selectorLabel="Shell"
  options={[
    {
      label: "Unix shell",
      code: "mkdir -p src/content/lessons",
      language: "bash",
    },
    {
      label: "PowerShell",
      code: "New-Item -ItemType Directory -Force -Path src/content/lessons",
      language: "powershell",
    },
    {
      label: "Command Prompt",
      code: "mkdir src\\content\\lessons",
      language: "bat",
    },
  ]}
/>

## Creating empty files with `touch`

To create an empty file or update its timestamp:

<CommandBlock
  metaLabel="Terminal"
  selectorLabel="Shell"
  options={[
    { label: "Unix shell", code: "touch README.md", language: "bash" },
    {
      label: "PowerShell",
      code: "New-Item README.md -ItemType File",
      language: "powershell",
    },
    {
      label: "Command Prompt",
      code: "type nul > README.md",
      language: "bat",
    },
  ]}
/>

`touch` is the common Unix-style command. PowerShell and Command Prompt use different forms for the same basic job.

## Copying with `cp`

To copy a file:

<CommandBlock
  metaLabel="Terminal"
  selectorLabel="Shell"
  options={[
    {
      label: "Unix shell",
      code: "cp README.md README.backup.md",
      language: "bash",
    },
    {
      label: "PowerShell",
      code: "Copy-Item README.md README.backup.md",
      language: "powershell",
    },
    {
      label: "Command Prompt",
      code: "copy README.md README.backup.md",
      language: "bat",
    },
  ]}
/>

To copy directories recursively:

<CommandBlock
  metaLabel="Terminal"
  selectorLabel="Shell"
  options={[
    { label: "Unix shell", code: "cp -r src src-backup", language: "bash" },
    {
      label: "PowerShell",
      code: "Copy-Item src src-backup -Recurse",
      language: "powershell",
    },
    {
      label: "Command Prompt",
      code: "xcopy src src-backup /e /i",
      language: "bat",
    },
  ]}
/>

## Removing with `rm`

To remove a file:

<CommandBlock
  metaLabel="Terminal"
  selectorLabel="Shell"
  options={[
    { label: "Unix shell", code: "rm notes.txt", language: "bash" },
    {
      label: "PowerShell",
      code: "Remove-Item notes.txt",
      language: "powershell",
    },
    {
      label: "Command Prompt",
      code: "del notes.txt",
      language: "bat",
    },
  ]}
/>

To remove a directory and its contents:

<CommandBlock
  metaLabel="Terminal"
  selectorLabel="Shell"
  options={[
    { label: "Unix shell", code: "rm -r old-folder", language: "bash" },
    {
      label: "PowerShell",
      code: "Remove-Item old-folder -Recurse",
      language: "powershell",
    },
    {
      label: "Command Prompt",
      code: "rmdir /s old-folder",
      language: "bat",
    },
  ]}
/>

<Warning title="rm deserves extra care">
  <p>Destructive terminal commands such as <code>rm</code>, <code>Remove-Item</code>, <code>del</code>, and <code>rmdir</code> deserve extra care. They often delete directly rather than sending content to a graphical trash folder. Read the path carefully before running them.</p>
</Warning>

## A practical rule

Creation and copying are usually reversible.

Removal is not always easy to undo.

That is why terminal users often move more slowly around `rm` than around `mkdir`, `touch`, `cp`, or `mv`.
