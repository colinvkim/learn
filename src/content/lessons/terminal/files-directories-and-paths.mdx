The terminal becomes much easier once you understand what commands are acting on.

## Files and directories

A **file** stores data.

A **directory** contains files and other directories.

Directories are also often called folders, but terminal commands usually use the word **directory**.

## A path identifies a location

A **path** tells the computer where a file or directory is located.

Examples:

```text
/Users/colin/Programming/learn
/Users/colin/Programming/learn/package.json
src/content/lessons
```

## Absolute paths

An **absolute path** starts from a fixed root location.

On macOS and Linux, absolute paths usually begin with `/`.

Example:

```text
/Users/colin/Programming/learn
```

An absolute path means the same location no matter where your current working directory is.

## Relative paths

A **relative path** is interpreted from the current working directory.

Example:

```text
src/content/lessons
```

That path only makes sense relative to where you currently are.

## `.` and `..`

Two path shortcuts matter immediately:

- `.` means the current directory
- `..` means the parent directory

Examples:

```text
./README.md
../package.json
```

## Why this matters for commands

When you run a command in the terminal, it often works on:

- the current directory
- a path you provide

That is why path understanding is not optional. It is the context that makes the commands meaningful.

<Note title="Relative paths are common in project work">
  <p>In real developer workflows, many commands use relative paths because you are already inside the project directory. That is why the current working directory matters so much.</p>
</Note>
