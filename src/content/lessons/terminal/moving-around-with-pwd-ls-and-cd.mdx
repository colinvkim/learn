Three commands form the basic navigation toolkit:

- `pwd`
- `ls`
- `cd`

On Windows, the exact commands depend on which shell you use. PowerShell and Command Prompt do not always use the same command names as Unix-style shells.

## `pwd` shows the current working directory

<CommandBlock
  metaLabel="Terminal"
  selectorLabel="Shell"
  options={[
    { label: "Unix shell", code: "pwd", language: "bash" },
    { label: "PowerShell", code: "Get-Location", language: "powershell" },
    { label: "Command Prompt", code: "cd", language: "bat" },
  ]}
/>

This prints the full path of the directory you are currently in.

That matters because many commands act relative to the current working directory.

## `ls` lists the contents of the current directory

<CommandBlock
  metaLabel="Terminal"
  selectorLabel="Shell"
  options={[
    { label: "Unix shell", code: "ls", language: "bash" },
    { label: "PowerShell", code: "Get-ChildItem", language: "powershell" },
    { label: "Command Prompt", code: "dir", language: "bat" },
  ]}
/>

This shows the files and directories in the current location.

A common variation is:

<CommandBlock
  metaLabel="Terminal"
  selectorLabel="Shell"
  options={[
    { label: "Unix shell", code: "ls -la", language: "bash" },
    { label: "PowerShell", code: "Get-ChildItem -Force", language: "powershell" },
    { label: "Command Prompt", code: "dir /a", language: "bat" },
  ]}
/>

That usually shows more detail, including hidden files.

## `cd` changes the current directory

To move into a directory:

<CommandBlock code="cd src" language="bash" metaLabel="Terminal" />

To move up to the parent directory:

<CommandBlock code="cd .." language="bash" metaLabel="Terminal" />

To move to a specific absolute path:

<CommandBlock
  metaLabel="Terminal"
  selectorLabel="Platform"
  options={[
    {
      label: "macOS or Linux",
      code: "cd /Users/colin/Programming/learn",
      language: "bash",
    },
    {
      label: "Windows",
      code: "cd C:\\Users\\colin\\Programming\\learn",
      language: "powershell",
    },
  ]}
/>

## Why these three commands matter together

They answer three practical questions:

- `pwd`: Where am I?
- `ls`: What is here?
- `cd`: How do I move somewhere else?

That is the basic navigation loop of terminal work.

<Tip title="Use pwd when context is unclear">
  <p>If a command behaves unexpectedly, checking the current working directory is often the fastest way to understand why. That might mean <code>pwd</code>, or <code>cd</code> with no argument in Command Prompt.</p>
</Tip>
