Navigation work usually comes down to three questions:

- Where am I?
- What is here?
- How do I move somewhere else?

Different shells answer those questions with different command names, so this lesson is organized by task rather than by treating Unix names as the baseline.

## Show the current working directory

<ShellCommandSet
  title="Show where you are"
  intro="This prints the full path of the directory you are currently in."
  unix={`pwd`}
  powershell={`Get-Location`}
  cmd={`cd`}
/>

That matters because many commands act relative to the current working directory.

## List the contents of the current directory

<ShellCommandSet
  title="List files and directories"
  intro="Each shell has a command for inspecting the current location. The examples below also include a common way to reveal hidden items."
  unix={`ls
ls -la`}
  powershell={`Get-ChildItem
Get-ChildItem -Force`}
  cmd={`dir
dir /a`}
/>

## Change directories

<ShellCommandSet
  title="Move into, out of, and across directories"
  intro="The underlying idea is the same in every shell: give the shell a path, and it changes the current working directory."
  unix={`cd src
cd ..
cd /Users/colin/Programming/learn`}
  powershell={`Set-Location src
Set-Location ..
Set-Location C:\\Users\\colin\\Programming\\learn`}
  cmd={`cd src
cd ..
cd C:\\Users\\colin\\Programming\\learn`}
/>

## Why these three commands matter together

They answer three practical questions:

- show the current directory
- inspect what is in it
- move to another directory

That is the basic navigation loop of terminal work.

<Tip title="PowerShell often supports Unix-style aliases">
  <p>PowerShell commonly accepts aliases such as <code>pwd</code>, <code>ls</code>, and <code>cd</code>. This course shows canonical PowerShell names when they differ, so the command meaning stays clear even if the aliases are available.</p>
</Tip>
